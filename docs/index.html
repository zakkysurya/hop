hop

SSH Project Launcher & Directory Jumper

Lompat langsung ke server dan direktori project favorit Anda — tanpa history | grep lagi.

▶ Get Started 🐙 View on GitHub

Masalah yang Sering Dihadapi

Menghafal IP, nama user, dan direktori project di setiap server? Itu masa lalu.

Dulu:
$ history | grep <ip>
$ !<no>
$ history | grep <project>
$ !<no>
Sekarang:
$ hop <alias> ✨

Fitur Utama

Dirancang untuk mempercepat workflow SSH development Anda sehari-hari.

📂

Satu server bisa punya banyak direktori project, masing-masing dengan alias sendiri.

🚀

Auto Jumper

Otomatis cd ke path tujuan begitu SSH berhasil terhubung.

📋

Output hop list & hop path-list mudah dibaca sekilas.

🛠

Manajemen Interaktif

add, edit, remove, path-add, path-remove — semua dipandu prompt.

🔄

Migrasi Otomatis

Config lama otomatis dicadangkan & dikonversi ke skema terbaru.

⌨

Autocomplete Pintar

Tab-completion untuk command, alias host, dan alias path di Bash.

Konfigurasi

File config.yaml dibuat otomatis saat hop pertama kali dijalankan.

Lokasi config:

OSLokasi
Linux / macOS~/.config/hop/config.yaml
Windows%HOME%\.config\hop\config.yaml

FieldKeterangan
aliasNama unik untuk memanggil host di CLI
hostIP atau hostname server SSH
userUsername login SSH
portPort SSH (default 22)
identity_filePath ke SSH private key (opsional)
paths[].aliasNama singkat direktori
paths[].pathPath tujuan di server
paths[].commandPerintah default setelah cd (opsional)
config.yaml 
hosts:
  - alias: dev-projek
    host: xx.xx.xx.xx
    user: root
    port: 22
    paths:
      - alias: projek1
        path: /var/www/html/projek1
      - alias: projek2
        path: /var/www/html/projek2
        command: php artisan serve

Penggunaan

Cara cepat terhubung ke server dan menjalankan command remote.

🚀 Koneksi Cepat

Terhubung ke host dan otomatis masuk ke path pertama, atau pilih path spesifik.

hop dev-projek

hop dev-projek projek2

⚡ Override Command

Tentukan command sendiri setelah terhubung, menggantikan command default di config.

hop dev-projek projek2 -- htop

🔎 Hop Doctor

Cek koneksi ke semua host terdaftar sekaligus untuk memastikan server aktif.

hop doctor

⚙ Hop Exec (Non-Interaktif)

Jalankan perintah langsung di server tanpa masuk sesi shell. Cocok untuk automasi.

hop exec dev-projek -- echo "halo"

hop exec dev-projek projek1 -- ls -l

Semua Perintah

Daftar lengkap command yang tersedia di hop.

CommandFungsi
hop listLihat semua host terdaftar
hop addTambah host baru (interaktif)
hop edit <host>Ubah detail host
hop remove <host>Hapus host
hop doctorCek koneksi ke semua host
hop exec <host> -- <cmd>Jalankan command non-interaktif
hop path-list [<host>]Lihat semua path (semua host / spesifik)
hop path-add <host>Tambah path baru ke host
hop path-edit <host> <path>Ubah detail path
hop path-remove <host> <path>Hapus path dari host
hop helpTampilkan pesan bantuan

Instalasi

Single binary, tanpa dependensi eksternal yang berat.

🐧 Linux / macOS

Prasyarat: Go 1.20+ (go version) dan binary ssh sudah terpasang.

Terminal 
# 1. Clone & compile
git clone https://github.com/zakkysurya/hop.git
cd hop
go build -o hop .

# 2. Install ke PATH
cp hop ~/.local/bin/
# atau untuk seluruh user:
sudo cp hop /usr/local/bin/

# 3. Verifikasi
hop help

🪵 Windows

Prasyarat: Go compiler, OpenSSH Client aktif (default di Windows 10/11), PowerShell / Windows Terminal.

PowerShell 
# 1. Compile
go build -o hop.exe .

# 2. Install ke folder khusus
New-Item -ItemType Directory -Force -Path "C:\bin"
Copy-Item hop.exe -Destination "C:\bin\hop.exe"

# 3. Tambahkan C:\bin ke PATH
# (Start Menu → "Edit the system environment variables")

# 4. Verifikasi
hop.exe help

⚠ Penting: hop mencari config lewat variabel HOME. Di Windows, tambahkan environment variable: HOME = C:\Users\NamaUserAnda.

Autocomplete (Tab) di Bash

Aktifkan sekali, pakai selamanya.

Cara Aktifkan

Terminal 
hop completion bash >> ~/.bashrc
source ~/.bashrc

Cara Pakai

  1. hop <Tab><Tab> — daftar semua command & host alias
  2. hop dev-<Tab> — otomatis lengkap jadi hop dev-projek
  3. hop dev-projek <Tab> — daftar path-alias milik host tersebut
Demo Autocomplete 
# Tekan Tab untuk melihat opsi:
$ hop <Tab>
list  add  edit  remove  path-list
path-add  path-edit  path-remove  doctor
dev-projek  prod-server  staging

# Lengkapi host + path:
$ hop dev-projek <Tab>
projek1  projek2

# Override command:
$ hop dev-projek projek2 -- <Tab>
htop  vim  nano  docker  git
